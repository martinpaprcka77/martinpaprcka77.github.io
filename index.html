🧭 Done: merged into one repo

Two repos (dotfiles-powershell, dotfiles-tools) used to exist for independent release cycles, but that created real cross-repo coupling — menu items calling functions that only existed in the other repo, guarded by Invoke-IfAvailable, and two independent $env:DOTFILES_TOOLS/$env:DOTFILES_PWSH sources of truth that could drift apart. Merged into this repo (profile/ + toolkit/ subfolders) instead of a new repo, so the portal keeps the root martinpaprcka77.github.io URL — no subpath tradeoff. The old repos remain on GitHub with a README pointer here (no repo-archive tooling was available to do more than that). A future split back into 2 repos remains a documented option in the roadmap, not a plan.

📄 dotfiles-powershell + 🔧 dotfiles-tools → 📦 this repo
